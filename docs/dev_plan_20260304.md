# synneu.io 开发计划

> 日期：2026-03-04
> 版本：V1.0 规划报告

---

## 一、项目概述

**synneu.io** 是一款集成 Git 版本管理与 AI Agent（ZeroClaw）的多模态笔记应用，目标平台为 Android 和 iOS。

核心价值：
- 多模态输入（文本 + 语音）
- 本地文件为中心的数据存储
- Git 驱动的版本管理与多项目同步
- AI Agent 增强的内容处理能力

---

## 二、技术选型

| 模块 | 技术方案 | 说明 |
|------|----------|------|
| 移动端框架 | Flutter | 统一支持 Android / iOS，单一代码库 |
| 语言 | Dart | Flutter 官方语言 |
| 语音识别 | `speech_to_text` 插件 | 本地或云端语音转文字 |
| 音频录制 | `record` 插件 | 跨平台音频录制 |
| Git 操作 | `libgit2` / `git2dart` | 本地 Git 仓库操作 |
| AI Agent | ZeroClaw（集成接口） | 文本理解、内容增强 |
| 本地存储 | 文件系统 + SQLite（元数据） | 笔记文件存储 |
| 状态管理 | Riverpod 或 Bloc | Flutter 状态管理 |

## 三、第一版功能范围（MVP）

### 3.1 功能清单

```
F1. 笔记创建与更新
  F1.1 文本输入 -> 直接保存/更新到文件
  F1.2 语音输入 -> 保存语音文件 -> 自动识别为文字 -> 保存/更新文本文件

F2. 项目（Git 仓库）管理
  F2.1 创建新 Git 项目（本地仓库初始化）
  F2.2 选择已有项目
  F2.3 将笔记文件放入指定项目
  F2.4 版本提交（commit）
  F2.5 远端同步（push / pull）
  F2.6 Git 配置管理（用户名、邮箱、远端地址）
```

### 3.2 功能优先级

| 优先级 | 功能 | 说明 |
|--------|------|------|
| P0 | F1.1 文本笔记创建/更新 | 核心基础功能 |
| P0 | F2.1/F2.2 项目创建与选择 | 文件归档依赖 |
| P0 | F2.3/F2.4 文件放入项目并 commit | Git 版本管理基础 |
| P1 | F1.2 语音输入 | 多模态核心 |
| P1 | F2.5 远端同步 | 数据安全与多设备 |
| P2 | F2.6 Git 配置管理 | 体验完善 |

## 四、开发阶段划分

### 阶段一：基础架构搭建（第 1-2 周）

**目标：** 搭建 Flutter 工程骨架，打通文件读写链路。

- [ ] 初始化 Flutter 项目，配置 Android / iOS 构建环境
- [ ] 设计目录结构（feature-first 或 layer-first）
- [ ] 集成状态管理库（Riverpod 推荐）
- [ ] 实现本地文件系统读写工具类
- [ ] 设计数据模型：笔记（Note）、项目（Project）
- [ ] 搭建基础导航路由

**交付物：** 可运行的空白 Flutter 工程，支持文件读写。

---

### 阶段二：文本笔记功能（第 3-4 周）

**目标：** 完成 P0 文本笔记的创建与更新流程。

- [ ] 设计并实现笔记编辑页面 UI
- [ ] 实现"创建笔记"流程：输入标题 + 内容 -> 写入本地文件
- [ ] 实现"更新笔记"流程：读取文件 -> 编辑 -> 覆盖写入
- [ ] 实现笔记列表页（展示项目内所有笔记文件）
- [ ] 文件命名规则设计（如时间戳 + 标题）

**交付物：** 可完成文本笔记创建与更新的可运行 APP。

---

### 阶段三：Git 项目管理（第 5-7 周）

**目标：** 集成 Git，实现项目创建、选择、版本提交。

- [ ] 集成 `libgit2` 或 `git2dart`，封装 Git 操作接口
- [ ] 实现本地 Git 仓库初始化（创建项目）
- [ ] 实现项目列表与选择功能
- [ ] 实现笔记文件归入指定项目（文件复制/移动 + git add）
- [ ] 实现版本提交（git commit，自动生成 commit message）
- [ ] 实现 Git 配置页（用户名、邮箱）

**交付物：** 笔记可与 Git 项目绑定并进行版本管理。

---

### 阶段四：语音输入功能（第 8-9 周）

**目标：** 完成语音录制与语音转文字链路。

- [ ] 集成 `record` 插件，实现音频录制与保存（.m4a / .wav）
- [ ] 集成 `speech_to_text` 插件，实现实时或离线语音识别
- [ ] 语音转写结果写入文本文件，与文本笔记流程打通
- [ ] UI：录音按钮、录音状态展示、转写结果预览与编辑

**交付物：** 支持语音输入并自动转写为文字笔记。

---

### 阶段五：远端 Git 同步（第 10-11 周）

**目标：** 支持 push / pull 同步远端仓库。

- [ ] 实现 git remote 配置（添加/修改远端地址）
- [ ] 实现 git push（支持 SSH Key 或 HTTPS Token 认证）
- [ ] 实现 git pull / fetch + merge
- [ ] 冲突提示与处理（V1 以提示为主，不做复杂合并 UI）
- [ ] 同步状态反馈（进度、成功、失败）

**交付物：** 笔记数据可同步至远端 Git 仓库（GitHub / Gitee 等）。

---

### 阶段六：集成测试与发布准备（第 12 周）

**目标：** 质量保障，准备应用商店发布。

- [ ] 端到端功能测试（Android + iOS 真机）
- [ ] 权限申请与处理（麦克风、文件存储）
- [ ] 应用图标、启动图配置

## 六、关键风险与应对

| 风险 | 影响 | 应对措施 |
|------|------|----------|
| `libgit2` / `git2dart` 在移动端兼容性不稳定 | 高 | 提前做技术验证 POC；备选方案：通过 Flutter 调用平台原生 Git CLI |
| 语音识别精度不足（方言、噪声） | 中 | 先接入系统级语音识别，后期可集成 ZeroClaw AI 增强 |
| iOS 文件沙盒限制导致 Git 操作路径受限 | 高 | 使用 App Documents 目录，明确路径策略 |
| Git push/pull 网络认证复杂（SSH/HTTPS） | 中 | V1 优先支持 HTTPS Token，SSH 后续版本迭代 |
| ZeroClaw AI Agent 接口稳定性未知 | 中 | V1 AI 功能作为可选增强，核心功能不依赖 AI |
